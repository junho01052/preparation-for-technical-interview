Array
메모리에서 연속적으로 할당되는 요소들의 집합입니다
배열은 O(1) 시간 복잡도로 인덱스에 따른 요소에 접근할 수 있습니다
하지만 배열의 크기는 고정되어 있어, 동적으로 크기를 변경할 수 없습니다

사용법
일반적으로 배열은 정적 데이터를 저장하거나 데이터의 크기가 변하지 않을 때 사용됩니다
인덱스를 통해 빠르게 데이터에 접근해야 할 경우 유용합니다

LinkedList

각 노드가 데이터와 포인터를 가지고 하나씩 연결되어 있는 선형적인 자료 구조입니다
연결 리스트는 동적으로 크기가 변경될 수 있으며, 삽입과 삭제가 O(1) 시간 복잡도로 가능합니다
하지만 특정 인덱스의 데이터에 접근하려면 O(n) 시간이 걸립니다

사용법
연결 리스트는 데이터의 삽입과 삭제가 빈번하게 일어나며, 데이터의 크기가 동적으로 변할 때 유용합니다
중간 데이터의 추가나 삭제가 자주 발생할 경우 사용됩니다

Array vs LinkedList
Array: 랜덤한 인덱스에 있는 요소에 빠르게 접근해야 하거나 메모리를 효율적으로 사용해야 할 때 사용합니다
LinkedList: 중간에 요소를 추가하거나 삭제할 때, 또는 크기가 자주 변경될 때 사용합니다

각 자료 구조는 상황과 필요에 따라 선택되어야 하며, 그에 따라 효율성이 달라질 수 있습니다
