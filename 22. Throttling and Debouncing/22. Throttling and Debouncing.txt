Throttling

개념:
Throttling은 일정 주기로 이벤트를 처리하는 방법입니다
예를 들어, 스크롤 이벤트가 발생할 때마다 로직을 실행한다면 브라우저는 매우 빠르게 여러 번의 이벤트를 처리해야 할 것입니다
쓰로틀링을 적용하면, 예를 들어 100ms마다 한 번씩만 이벤트를 처리하도록 할 수 있습니다

사용하는 이유:
브라우저의 과부하를 막기 위해
불필요한 API 호출을 줄이기 위해
사용자 경험을 향상시키기 위해 (스크롤할 때마다 로딩이 일어나는 것을 방지)

Debouncing

개념:
Debouncing은 일정 시간 동안 같은 이벤트가 여러 번 발생할 경우 마지막 이벤트만을 처리하는 방법입니다
예를 들어, 검색창에 텍스트를 입력할 때마다 API를 호출한다면,
사용자가 'apple'을 입력하려고 할 때 'a', 'ap', 'app', 'appl', 'apple' 이렇게 5번의 API 호출이 발생할 수 있습니다
디바운싱을 적용하면, 사용자가 입력을 완료하고 일정 시간 동안 다른 입력이 없을 때 한 번만 API를 호출합니다

사용하는 이유:
불필요한 API 호출을 줄이기 위해
사용자 경험을 향상시키기 위해 (입력 중에 계속 로딩이 일어나는 것을 방지)
서버 리소스를 효율적으로 사용하기 위해

차이점
Throttling: 일정 시간 동안 이벤트를 일정 횟수만큼만 실행시키는 것, 스크롤 이벤트, 리사이즈 이벤트, 마우스 무브 이벤트 등에 주로 사용됩니다
Debouncing: 일정 시간 동안 발생한 이벤트 중 마지막 이벤트만을 실행시키는 것, 검색창, 폼 유효성 검사, 버튼 중복 클릭 방지 등에 주로 사용됩니다
